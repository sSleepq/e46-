# 1. Теоретическая часть

## Хранимая процедура (Stored Procedure)

**Хранимая процедура** — это набор SQL-команд, сохранённый в базе данных, который можно вызывать многократно.

Процедуры позволяют:

* автоматизировать выполнение операций
* уменьшить дублирование кода
* повысить безопасность
* инкапсулировать бизнес-логику

---

## Синтаксис создания процедуры

```sql id="l8k2df"
CREATE PROCEDURE procedure_name()
BEGIN
    -- SQL команды
END;
```

---

## Вызов процедуры

```sql id="p2x9wq"
CALL procedure_name();
```

---

## Процедура с параметрами

```sql id="t9v4nm"
CREATE PROCEDURE get_students_by_city(IN city_name VARCHAR(50))
BEGIN
    SELECT * FROM students WHERE city = city_name;
END;
```

Вызов:

```sql id="v7r2pl"
CALL get_students_by_city('Москва');
```

---

## Входные и выходные параметры

* `IN` — входной параметр
* `OUT` — выходной параметр
* `INOUT` — комбинированный

Пример:

```sql
CREATE PROCEDURE count_students(OUT total INT)
BEGIN
    SELECT COUNT (*) INTO total FROM students
END;
```

---

## Использование условий

```sql id="y6m1rt"
CREATE PROCEDURE check_grade(IN g INT)
BEGIN
    IF g >= 4 THEN
        SELECT 'Хорошо';
    ELSE
        SELECT 'Нужно улучшить';
    END IF;
END;
```

---

## Удаление процедуры

```sql id="x4q7bz"
DROP PROCEDURE procedure_name;
```
