# 1. Теоретическая часть

## Индекс (INDEX)

**Индекс** — это структура данных, которая ускоряет поиск строк в таблице.

Без индекса база данных выполняет **полный перебор строк (FULL SCAN)**.
С индексом поиск происходит значительно быстрее, особенно в больших таблицах.

---

## Зачем нужны индексы

Индексы используются для:

* ускорения `SELECT` запросов
* ускорения поиска по `WHERE`
* ускорения соединений (`JOIN`)
* ускорения сортировки (`ORDER BY`)

---

## Недостатки индексов

* замедляют `INSERT`, `UPDATE`, `DELETE`
* занимают дополнительную память
* требуют грамотного использования

---

## Синтаксис создания индекса

```sql id="r7mghd"
CREATE INDEX index_name
ON table_name (column_name);
```

---

## Пример

```sql id="4g2v2r"
CREATE INDEX idx_student_name
ON students (name);
```

---

## Составной индекс

Индекс может включать несколько колонок:

```sql id="g0l7me"
CREATE INDEX idx_group_city
ON students (group_name, city);
```

---

## Удаление индекса

```sql id="3eb2kq"
DROP INDEX index_name;
```

---

## Уникальный индекс

```sql id="f8q3c9"
CREATE UNIQUE INDEX idx_unique_email
ON users (email);
```

Гарантирует уникальность значений.
