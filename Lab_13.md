# 1. Теоретическая часть

## Триггер (TRIGGER)

**Триггер** — это специальный объект базы данных, который автоматически выполняется при наступлении определённого события:

* `INSERT` (добавление данных)
* `UPDATE` (изменение данных)
* `DELETE` (удаление данных)

---

## Когда используются триггеры

Триггеры применяются для:

* автоматической проверки данных
* ведения журналов (логирования)
* контроля целостности данных
* автоматических вычислений

---

## Синтаксис создания триггера

```sql id="w8r1k3"
CREATE TRIGGER trigger_name
AFTER INSERT ON table_name
FOR EACH ROW
BEGIN
    -- действия
END;
```

---

## Типы триггеров

### По времени выполнения:

* `BEFORE` — до выполнения операции
* `AFTER` — после выполнения операции

### По событию:

* `INSERT`
* `UPDATE`
* `DELETE`

---

## Специальные таблицы

В триггерах используются:

* `NEW` — новые данные
* `OLD` — старые данные

Пример:

```sql id="r8y3kp"
CREATE TRIGGER check_grade
BEFORE INSERT ON grades
FOR EACH ROW
BEGIN
    IF NEW.grade < 1 OR NEW.grade > 5 THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'Оценка должна быть от 1 до 5';
    END IF;
END;
```

---

## Пример логирования

```sql id="k3p9mv"
CREATE TRIGGER log_insert
AFTER INSERT ON students
FOR EACH ROW
BEGIN
    INSERT INTO logs(action, description)
    VALUES ('INSERT', CONCAT('Добавлен студент: ', NEW.name));
END;
```

---

## Удаление триггера

```sql id="f2k9lm"
DROP TRIGGER trigger_name;
```
