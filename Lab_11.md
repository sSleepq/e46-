# 1. Теоретическая часть

## Представление (VIEW)

**VIEW (представление)** — это виртуальная таблица, которая формируется на основе результата SQL-запроса.

Представление **не хранит данные**, а хранит только **SQL-запрос**, который выполняется при обращении к нему.

Представления используются для:

* упрощения сложных запросов
* повышения безопасности данных
* ограничения доступа к таблицам
* повторного использования SQL-запросов

---

## Синтаксис создания VIEW

```sql
CREATE VIEW view_name AS
SELECT column1, column2
FROM table_name
WHERE condition;
```

---

## Пример

Создадим представление со списком студентов и их оценок.

```sql
CREATE VIEW student_grades AS
SELECT students.name, courses.course_name, grades.grade
FROM students
JOIN grades ON students.id = grades.student_id
JOIN courses ON courses.id = grades.course_id;
```

После этого можно выполнять запрос:

```sql
SELECT * FROM student_grades;
```

---

## Изменение представления

```sql
CREATE OR REPLACE VIEW view_name AS
SELECT ...
```

---

## Удаление представления

```sql
DROP VIEW view_name;
```
